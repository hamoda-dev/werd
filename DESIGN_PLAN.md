## السياق (لماذا هذا التطبيق)

تطبيق **أذكار** يعمل بالكامل **محلياً على الجهاز بلا أي اتصال بالشبكة**.
الهدف: مساعدة المستخدم على المداومة على أذكار الصباح والمساء، مع قابلية
إضافة أوراد خاصة به، وتنبيهات تحفّزه على الاستمرار.

- المحتوى الأساسي: أذكار الصباح (١٣) والمساء (١٢) من «الخلاصة الحسناء» — مُستخرَجة في
  [src/data/adhkar.json](src/data/adhkar.json).
- التخصيص: المستخدم يضيف أوراده الخاصة (نص + عدد تكرار).
- Onboarding بسيط: نسأل عن الاسم فقط، ويُحفظ كل شيء محلياً.
- تنبيهات تحفيزية يومية (صباح/مساء) قابلة للضبط.

## القرارات المعتمدة

| القرار | الاختيار |
|--------|----------|
| اتجاه الواجهة | **ب — تحفيزي ومرِح** (خلفية خضراء داكنة، streak كبطل، نقاط/مستوى) |
| المنصة | Expo SDK 55 · React Native 0.83 · TypeScript · expo-router |
| التخزين | محلي فقط — `@react-native-async-storage/async-storage` (لا شبكة) |
| الإشعارات | `expo-notifications` — إشعارات محلية مجدولة (بلا push/سيرفر) |
| الاتجاه | RTL إجباري عبر `I18nManager.forceRTL(true)` |
| الخطوط | IBM Plex Sans Arabic (واجهة/أرقام) + Amiri (نص الأذكار) عبر `expo-font` |
| الأرقام | عربية شرقية (٠-٩) عبر دالة تحويل |
| الحركات | `react-native-reanimated` (مثبّت مسبقاً) لحلقة المسبحة والاحتفال |
| إدارة الحالة | React Context + hooks مخصّصة فوق AsyncStorage (بلا مكتبة إضافية) |

## القرارات المفتوحة (توصياتي — قابلة للتعديل)

1. **عدد «أعوذ بكلمات الله التامات» (المساء):** وضعته **مرة واحدة** كما في الـPDF،
   والمشهور في السُّنّة **ثلاث مرات**. ← أحتاج تأكيدك.
2. **مكان الأوراد الخاصة:** توصيتي = قسم «أورادي» في الشاشة الرئيسية تحت بطاقتي
   الصباح/المساء، مع زر «+ إضافة وِرد». كل وِرد يفتح **نفس شاشة المسبحة** (إعادة استخدام
   المكوّن الأساسي). الإدارة (إضافة/تعديل/حذف) عبر نموذج بسيط.
3. **نطاق الإصدار الأول (v1):** أركّز على القلب (streak + الصباح/المساء + المسبحة +
   الأوراد + التنبيهات)، وأؤجّل التوسّع الكامل (شارات/تحديات/تقويم إحصائي) إلى v2.

## نموذج البيانات (AsyncStorage)

```ts
// مفاتيح التخزين
settings   = { name: string, remindersEnabled: bool, morningTime: "07:00", eveningTime: "18:30" }
progress   = { [date: "YYYY-MM-DD"]: { morningDone: bool, eveningDone: bool, completedIds: string[] } }
streak     = { current: number, longest: number, lastCompletedDate: string }
score      = { points: number, level: number }
customAwrad= [ { id, title, text, count, createdAt } ]
```

الأذكار المدمجة تأتي من `adhkar.json` (للقراءة فقط). كل عنصر يتبع نفس الشكل
`{ id, text, count, note?, title? }` ليتوحّد مكوّن العرض.

## التنبيهات (expo-notifications)

- إشعاران محليان يوميان مجدولان: الصباح (افتراضي 07:00) والمساء (افتراضي 18:30).
- نبرة لطيفة محفّزة: «صباح النور 🌤 — لا تنسَ أذكار الصباح» / «لا تكسر سلسلتك 🔥».
- طلب الإذن عند تفعيل التنبيهات أول مرة. إعادة الجدولة عند تغيير الوقت.
- لا حاجة لأي سيرفر أو push token — كله محلي.

## خطة التنفيذ على مراحل

**المرحلة 0 — الأساس**
1. تثبيت: `@react-native-async-storage/async-storage`, `expo-notifications`.
2. تحميل الخطوط (IBM Plex Sans Arabic + Amiri) عبر `expo-font` + ملف splash/font gate.
3. تفعيل RTL في `app/_layout.tsx` + `app.json` (locales). إضافة `src/theme/tokens.ts`.
4. مساعدات: `toArabicNumerals(n)`, `useStorage<T>(key)`.

**المرحلة 1 — البيانات والحالة**
5. `src/store/` : Context + hooks (`useSettings`, `useProgress`, `useStreak`, `useCustomAwrad`).
6. منطق السلسلة والنقاط (تحديث عند إكمال تصنيف).

**المرحلة 2 — القلب (المسبحة + القوائم)**
7. مكوّن `TasbihCounter` (Reanimated: حلقة التقدّم + احتفال wd-ring).
8. شاشة `session/[category]` (تنقّل بين الأذكار) + شاشة `list/[category]`.

**المرحلة 3 — الواجهات الرئيسية**
9. Onboarding (الاسم). 10. Home (اتجاه ب). 11. تبويبات الإنجازات/الملف.

**المرحلة 4 — الأوراد الخاصة والتنبيهات**
12. نماذج إضافة/تعديل/حذف الأوراد. 13. جدولة التنبيهات + شاشة الإعدادات.

**المرحلة 5 — اللمسات**
14. حركات الدخول، حالات فارغة/خطأ، أيقونة التطبيق وsplash من شعار `wird-logo.png`.

## معايير النجاح (v1)

- يفتح التطبيق بلا إنترنت ويعمل كل شيء offline.
- المستخدم يكمل أذكار الصباح عبر المسبحة، فترتفع السلسلة وتُحفظ بعد إغلاق التطبيق.
- يضيف وِرداً خاصاً ويشغّله بنفس المسبحة.
- يصله تنبيه صباح/مساء في الوقت المحدد.
- الواجهة RTL سليمة بالأرقام العربية والخطين الصحيحين.
